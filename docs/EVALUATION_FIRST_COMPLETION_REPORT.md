## 1. 本轮目标与范围

本轮将产品开发优先级收紧到“评价功能可用”，同时保证账号、项目与上传等基础能力不被破坏。

- 优先完成：11 模块复核、得分调整、完整性校验、评分锁定、版本历史、最新评分回显。
- 保持可用：注册、登录、会话恢复、个人资料、通知偏好、项目创建、PDP 上传。
- 主动暂缓：品牌资产库、企业 DAM、AI 生图、AI 页面生成。
- 暂缓能力统一以灰色禁用状态展示，并明确标注“暂未开放”。

## 2. 当前完整交互 Flow

```text
注册 / 登录
  ↓
独立首页
  ├─ 选择最近项目 ───────────────→ 项目总览
  └─ 选择项目 / 新建项目 → 上传 PDP → 诊断进度
                                      ↓
评分确认（11 个模块逐项复核）
  ├─ 调整弱 / 中 / 强 → 自动重算模块得分、总分和星级
  ├─ 未确认完整 → 禁止锁定
  └─ 全部确认 → 后端校验 → 创建评分版本 vN
                                      ↓
评分记录 → 查看版本、操作者、时间与 11 项明细
                                      ↓
项目总览 / 评分诊断读取最新锁定版本
                                      ↓
优化路线（可用）→ 品牌资产 / AI 创作（灰显暂缓）
```

## 3. 已完成的评价功能

### 前端

- “评分诊断”增加“复核并创建新评分版本”入口。
- 评分确认页支持选择模块并调整成熟度：弱 = 0、中 = 0.5、强 = 1。
- 成熟度变化后实时计算模块得分、总分和整体星级。
- 支持逐项确认，也支持“一键确认全部 AI 初评”后再锁定。
- 未完成全部 11 项确认时，“确认并锁定评分”不可点击。
- 保存失败会在当前页显示错误，不会错误跳转。
- 新增“评分记录”页面，显示评分版本列表与完整模块明细。
- 项目总览和评分诊断读取最新锁定版本，评分、星级、成熟度数量和模块得分保持同步。

### 后端

- 新增 `DiagnosisVersion` 数据模型：
  - 项目
  - 自动递增版本号
  - 总分
  - 整体星级
  - 11 模块 JSON 明细
  - 锁定状态
  - 操作者
  - 创建时间
- 新增 `GET /api/diagnoses/?project_id=<id>` 获取评分历史。
- 新增 `POST /api/diagnoses/` 创建锁定评分版本。
- 后端校验：
  - 必须正好提交 11 个模块。
  - 每个模块必须经过确认。
  - 成熟度只能为弱 / 中 / 强。
  - 得分不能小于 0 或超过模块满分。
  - 提交总分必须等于模块得分总和。
  - 同一项目版本号唯一，并在事务内递增。

## 7. 当前限制与下一步建议

1. 当前“诊断进度”和“AI 初评证据”仍是本地模拟，尚未连接 OCR、页面切片或视觉理解服务。
2. 评分规则已持久化结果，但模块规则本身仍写在前端常量中；下一步建议建立 `ScoringRule` / `ScoringModule` 后端模型并由 API 下发。
3. 证据图目前为示例项目图；下一步应保存每个模块对应的页面坐标、OCR 文本和证据截图。
4. 正式部署前需要移除 `csrf_exempt`，启用 CSRF 防护、生产 Cookie 策略、对象存储和权限隔离。
5. 资产库与 AI 生成只有在真实服务、任务状态和审核流程接入后再解除灰显。
