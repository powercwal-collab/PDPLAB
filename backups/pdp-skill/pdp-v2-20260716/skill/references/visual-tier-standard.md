# Visual Tier Standard

Use this file when diagnosing PDP visual quality, case positioning, material standards, layout direction, and whether a module should be judged weak, medium, or strong based on visual execution.

This file does not change the 11-module weights or star bands. It adds a visual-tier lens before scoring and makes material/layout quality part of module maturity judgment.

## Case Positioning

Before scoring, identify the case positioning:

| dimension | question | scoring use |
|---|---|---|
| Category | Is it shoes, apparel, socks, hats, bags, gear, beauty, electronics, or another category? | Defines the visual proof users expect |
| Product role | Is it a basic SKU, long-term seller, core style, new launch, flagship, collaboration, or campaign product? | Defines visual investment level |
| User decision risk | Is the main risk fit, comfort, safety, performance, styling, material, service, or value? | Defines priority modules |
| Visual tier | Is the page aiming for T1 standard conversion or T0 benchmark growth? | Defines material and layout threshold |
| Competitive reference | Is the reference sports fashion, lifestyle/street, premium sports, luxury crossover, or functional specialist? | Defines art direction |
| Page task | Is the page explaining, converting, launching, branding, educating, or supporting a campaign? | Defines narrative rhythm |

Do not score a page only by whether a module exists. A module must be judged against the case's expected visual tier.

Treat existence as effective existence. Layout containers, headings, repeated or decorative imagery, generic icons, and templated text do not qualify a module for `中` unless they carry product-specific information or visual evidence that answers a concrete purchase question.

## T2: Basic Material PDP

T2 is a basic material PDP. It is useful as a diagnostic baseline, not as the default target for mature PDP design.

T2 is common when a page only has white-background product photos, flat-lay images, basic model images, color thumbnails, and simple specifications.

T2 goal:

- Show what the product is.
- Confirm basic color, shape, SKU, and price.
- Provide minimal information for low-risk or low-investment products.

T2 typical signs:

| direction | typical execution |
|---|---|
| Material | White-background product images, flat-lay images, simple model images, color thumbnails |
| Information | Basic product name, color, size, price, material, and SKU |
| Layout | Listing-like vertical stacking, limited rhythm, limited decision guidance |
| Visual | Product is visible but not strongly attractive or explanatory |
| Conversion | Can support basic recognition, but usually cannot reduce deeper decision risk |

T2 in scoring:

- T2 does not automatically mean weak. If a module exists, it can still be `中`.
- T2 rarely supports `强` for high-risk, campaign, flagship, or category-education products.
- If the page only has T2 materials and lacks a corresponding module, score that module `弱`.
- If the page has T2 materials but no visual explanation, no scene, and no consumer-demand fit, score the module `中`.
- Exception: apply the hard module gates. Copy-only content scores `0` for `产品KV/封面故事`; white/gray-background model photos or front/back model views alone score `0` for `沉浸式购物/场景化`, even if those assets are polished.

## T1: Standard Conversion PDP

T1 is a standardized conversion PDP. It is suitable for regular products, long-term sellers, high-SKU operations, and pages where efficiency and clarity matter more than campaign expression.

T1 goal:

- Explain the product clearly.
- Reduce decision risk.
- Improve add-to-cart and conversion.
- Keep production repeatable.

T1 strong standard:

| direction | strong execution |
|---|---|
| Material | Clear product images, model/on-body images, detail images, basic scenario images, function graphics |
| Information | Selling points, size, material, fit, usage scenario, care, service, and recommendation are easy to find |
| Layout | Modules are ordered logically, one screen has one main task, scanning cost is low |
| Visual | Clean, focused, not decorative, not a plain data sheet |
| Conversion | Users can quickly judge whether the product fits their needs |

T1 medium signs:

- Information exists but is text-heavy.
- Product photos are clear but generic.
- Detail images show parts but do not explain benefits.
- Layout is tidy but lacks attraction or rhythm.
- Module order is complete but does not guide a decision.

T1 weak signs:

- Key modules are missing.
- Product images are too few, low quality, or not category-relevant.
- Size, fit, material, or use scenarios are absent.
- The page is mostly a plain product listing.

## T0: Benchmark Growth PDP

T0 is a benchmark growth PDP. It is suitable for flagship products, new launches, collaborations, high-price products, hero SKUs, campaign pages, and pages expected to become brand-level content assets.

T0 goal:

- Attract attention.
- Build product desire.
- Prove product value.
- Create brand memory.
- Support premium pricing, campaign traffic, and shareable content.

T0 strong standard:

| direction | strong execution |
|---|---|
| Material | Campaign-level KV, hero product visuals, motion/state images, lifestyle or sport scenes, premium detail shots |
| Information | Selling points are translated into scenarios, body movement, comparison, tests, diagrams, or product structures |
| Layout | Strong hierarchy, editorial rhythm, visual tension, meaningful white space, clear narrative progression |
| Visual | Creates a recognizable product world, brand mood, or category leadership signal |
| Conversion | Explains the product while raising perceived value and memorability |

T0 medium signs:

- Campaign intent exists but visuals are ordinary.
- Large images are used but do not explain a decision point.
- The page has style but lacks functional proof.
- The page has proof but lacks brand mood.
- Layout is visually active but does not guide reading.

T0 weak signs:

- The product is positioned as flagship/campaign, but the page uses only T1-level assets.
- Visuals are decorative and do not carry product meaning.
- No hero narrative, no strong scene, no memory point.
- The page cannot justify premium, launch, or campaign positioning.

## Material Direction

Judge materials by whether they answer category-specific purchase questions, not by quantity.

| material type | T1 requirement | T0 requirement |
|---|---|---|
| Product image | Clear white/light-background multi-angle images | Product becomes the visual hero through lighting, composition, scale, and texture |
| Detail image | Shows key parts such as outsole, fabric, zipper, brim, pocket, lining | Turns details into proof of technology, comfort, craft, durability, or premium value |
| Model/on-body image | Shows fit, size proportion, styling, and usage | Shows movement, attitude, lifestyle, sport state, or brand identity |
| Scenario image | Shows real use context | Builds product world, emotional context, and category leadership |
| Function graphic | Uses icons, diagrams, labels, or comparison to clarify benefit | Uses visual systems, motion, tests, cutaways, or structure logic to make proof memorable |
| Color/SKU image | Shows color and SKU differences | Uses color nicknames, styling logic, series matrix, or scenario mapping |
| Comparison image | Helps choose between versions or features | Turns comparison into visual decision guidance, not a dense table |
| Video/dynamic content | Optional, but should explain function when used | Should be a core proof tool for motion, structure, try-on, texture, or scene |

Material quality checks:

- Relevance: does the asset answer a real purchase question?
- Attraction: does it make users want to keep reading?
- Explanation: does it make a benefit easier to understand?
- Category fit: does it match category expectations?
- Tier fit: does it reach the visual level promised by T1 or T0 positioning?

## Layout And Art Direction

Judge layout by whether it turns materials into a decision journey.

| layout dimension | T1 standard | T0 standard |
|---|---|---|
| First screen | Product and core selling point are clear | Strong KV, product role, main claim, and visual memory point |
| Information hierarchy | Title, image, benefit, and proof are easy to scan | Strong contrast, editorial rhythm, and intentional tension |
| Module rhythm | One module answers one decision question | Page has narrative progression and emotional pacing |
| Image/text relation | Image explains text and text clarifies image | Image and text jointly create desire, proof, and brand mood |
| Composition | Cards, sections, tables, icons, and grids are clean | Large images, split screens, overlays, crops, scale contrast, and layered storytelling |
| Color | Brand/category color is consistent and readable | Color becomes a system for mood, category, product series, or campaign memory |
| Typography | Clear, readable, with visible emphasis | Has headline character, hierarchy, and brand tone |
| Long-page structure | Complete modules with low reading cost | Clear opening, proof development, detail deepening, decision support, and closing |

Layout medium signs:

- The page is organized but flat.
- The page is attractive but hard to scan.
- The layout repeats one pattern for too long.
- The page relies on text blocks or parameter tables without visual translation.
- The module exists but lacks visual rhythm, hierarchy, or focal point.

Apply these signs only after the module passes effective-existence and module-specific hard gates. A visually weak but useful product-specific module may be `中`; an attractive empty form or generic template remains `弱`.

For `产品KV/封面故事`, text without a product-led hero visual is not a medium KV; it is a missing qualifying KV and must be `弱`. For `沉浸式购物/场景化`, white/gray studio model photos and front/back views are product-display or fit assets, not qualifying scenes, and must be `弱` when no real context is present.

## Category-Specific Visual Cues

Use these as visual expectations when judging category fit.

### Shoes

Decision questions:

- Fit, cushioning, stability, grip, weight, breathability, durability, scene, styling.

T1 visual cues:

- Multi-angle product image.
- On-foot image.
- Outsole, midsole, upper, heel, toe, lining details.
- Size/fit guidance.
- Scenario image for running, training, basketball, outdoor, school, or daily wear.

T0 visual cues:

- Hero shoe composition.
- Sport motion or lifestyle scene.
- Cushioning, grip, energy return, support, or weather proof visualized.
- Colorway story and on-foot matrix.
- Product technology as a visual system.

### Apparel

Decision questions:

- Fit, silhouette, fabric, warmth/cooling, stretch, coverage, outfit, body type, care.

T1 visual cues:

- Front/back/side model photos.
- Size and model measurement.
- Fabric close-up.
- Wearing scenario.
- Color and outfit recommendation.

T0 visual cues:

- Editorial styling.
- Movement state.
- Fabric texture as hero detail.
- Lifestyle or sport mood.
- Series outfit matrix.

### Socks

Decision questions:

- Cushioning, compression, breathability, anti-slip, length, thickness, activity, comfort.

T1 visual cues:

- Product set image.
- Wearing image.
- Cushion/arch/toe/heel detail.
- Thickness and length comparison.
- Activity recommendation.

T0 visual cues:

- Graphic product rhythm.
- Touch/pressure/airflow visualized.
- Color series and outfit logic.
- Fun or lifestyle memory point.

### Hats

Decision questions:

- Sun protection, breathability, fit, head shape, brim, adjustability, sweatband, packability, styling.

T1 visual cues:

- Front/side/back/top images.
- On-head image.
- Brim, closure, sweatband, ventilation details.
- Size and head circumference guidance.

T0 visual cues:

- Outdoor or sport scenario.
- UPF/sun/rain/wind proof visualized.
- Face-shape and styling mood.
- Strong campaign or street visual.

### Bags And Accessories

Decision questions:

- Capacity, compartments, carrying comfort, durability, waterproofing, access, body proportion, sport/travel scenario.

T1 visual cues:

- Front/back/side/product details.
- On-body scale.
- Capacity and compartment diagram.
- Material and zipper details.
- Scenario image.

T0 visual cues:

- Gear-like hero scene.
- Packing/carrying story.
- Motion or outdoor context.
- Modular structure, capacity, and access visualized.
- Premium material and detail shots.

## How Visual Tier Enters Weak / Medium / Strong Scoring

Use visual tier as part of module maturity scoring.

## Unoptimized Baseline Cases

Use this section when reviewing unoptimized cases from category visual research boards. Do not put every unoptimized page into T2. First identify whether the page is truly basic material level, a weak T1 conversion page, or a T0 positioning mismatch.

### T2 Basic Baseline

Use `T2 Basic Baseline` when the page mainly shows the product but does not build a decision journey.

Typical signs:

- White-background product images.
- Flat-lay images.
- Basic model images without scenario meaning.
- Simple color/SKU listing.
- Basic parameters, price, material, or size table.
- No real scene, no proof translation, no visual narrative.

Scoring rule:

- Missing modules remain `弱`.
- Existing modules usually score `中` unless information is complete and the product is very low-risk.
- T2 basic baseline rarely supports `强` for kids, sport, apparel, shoes, bags, or other decision-heavy categories.

### T1-minus Current Baseline

Use `T1-minus Current Baseline` when the page has modules and information, but the design does not yet reach a strong T1 conversion standard.

Typical signs:

- Has modules such as size, detail, material, service, recommendation, or product comparison.
- Has long-page structure, but the rhythm feels like information stacking.
- Visual assets are generic, repetitive, or not attractive.
- Details show product parts but do not translate benefits.
- Scene/model images exist but do not explain use, fit, comfort, function, or styling.
- The page is readable, but it does not strongly guide decision-making.

Scoring rule:

- Do not mark these modules `弱` only because they are visually unoptimized.
- Mark related modules `中` when information or visual quality is not strong enough.
- Mark a module `强` only when information, material, layout, category fit, and consumer-demand fit all work together.

### T0 Positioning Gap

Use `T0 Positioning Gap` when the product or campaign should behave like T0, but the page execution stays at T1 or T2.

Typical signs:

- Product is a launch, hero SKU, flagship, collaboration, high-price item, or campaign product.
- Page has only standard product information or ordinary assets.
- Campaign resources are absent, or present but not connected to product proof.
- Visuals do not create brand memory, product desire, or category leadership.
- Layout does not have a clear narrative progression.

Scoring rule:

- A T0-positioned page with only T1/T2 execution should not be called `强` for visual modules.
- It can still score `中` for modules that exist and explain basic information.
- Use this gap to recommend campaign-level assets, stronger hero story, visual system, and proof translation.

### Baseline Classification Rule

| observed page | classify as | scoring meaning |
|---|---|---|
| Only white-background, flat-lay, basic model, color/SKU, parameters | T2 Basic Baseline | Basic material level |
| Has modules, but text-heavy, generic assets, weak rhythm, weak proof translation | T1-minus Current Baseline | Existing modules usually `中` |
| Product should be campaign/flagship, but page uses T1/T2 material | T0 Positioning Gap | Tier mismatch; do not over-score visual modules |

Use unoptimized baselines as negative references for improvement, not as proof that a module is missing.

### Weak

Mark a module `弱` when:

- There is no corresponding module.
- Users cannot obtain this type of purchase decision information.

Visual tier does not make a missing module medium. Missing is still weak.

### Medium

Mark a module `中` when the module exists but any of these are true:

- Information is shallow, scattered, generic, or hard to use.
- Visual assets are weak, unattractive, repetitive, or not explanatory.
- The module over-relies on information and lacks judgment of visual asset attractiveness.
- Materials do not match the category's decision questions.
- Layout is only information stacking and does not guide the decision.
- T2 materials exist but do not rise to the category's expected T1/T0 target.
- T1 positioning exists but basic material clarity, hierarchy, or scan efficiency is unstable.
- T0 positioning exists but the module only reaches T1-level material or layout.

Medium means the module exists, but either information quality, visual material quality, layout quality, category fit, or tier fit is not strong enough.

### Strong

Mark a module `强` when all are true:

- The module exists.
- Information answers concrete consumer purchase questions.
- Visual assets attract users and make the information easier to understand.
- Layout reduces reading cost or increases perceived value.
- Materials match the category decision logic.
- Execution reaches the expected T1 or T0 standard for the case positioning.

For T1, strong means efficient conversion. For T0, strong means benchmark growth and brand asset potential.

## Reporting Language

Use these phrases when explaining visual-tier judgment:

- "This module exists, but it only reaches information completion; visual attraction and proof translation are still medium."
- "The page is positioned like a T0 campaign product, but the current material system is closer to T1 standardized conversion."
- "The page currently sits at T2 material level: it shows the product, but does not yet translate materials into consumer decision proof."
- "The issue is not lack of content. The issue is that materials and layout do not yet match the product's case positioning."
- "T1 strong means efficient conversion. T0 strong means brand-level visual memory plus conversion proof."
- "Visual strength should be judged by whether it answers category purchase questions, not by whether the page looks decorated."
