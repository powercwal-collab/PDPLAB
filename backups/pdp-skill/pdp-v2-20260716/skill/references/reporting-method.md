# Reporting Method

Use this file when the user asks for a presentation, report logic, or stakeholder-facing explanation.

## Narrative Order

1. Conclusion: current score, star band, biggest gap.
2. Scoring standard: explain overall star vs module maturity.
3. Evidence: show module score table and visible page evidence.
4. Business logic: why it is this star, and why not the next star.
5. Actions: P0/P1/P2 improvement cases with required assets.
6. Validation: metrics to check after redesign.

## Key Language

Use concise consultant phrasing:

- "This is not a low-score page. It is a mature page with specific conversion gaps."
- "The problem is not page length; the problem is decision information density."
- "Strong visuals attract attention. Decision modules complete conversion."
- "Stars evaluate the whole page. Maturity evaluates each module."

## Validation Metrics

Tie actions to metrics:

| action | metric |
|---|---|
| Quick decision module | first-screen continuation, detail engagement |
| Size/fit module | size consultation, size-related returns |
| Proof module | add-to-cart rate, conversion rate |
| Dynamic demonstration | video play rate, module dwell time |
| Recommendation module | recommendation CTR, basket size |
| Service module | checkout completion, customer service questions |

## Kids Performance Validation

Use this when reporting kids footwear redesigns or main-image carousel optimization.

Overall PDP metrics:

| metric | what it validates |
|---|---|
| Dwell time | whether users are willing to continue reading after decision information is frontloaded |
| Interaction rate | whether carousel, size, color, video, or recommendation modules become more useful |
| Bounce rate | whether first-screen clarity reduces early exit |
| Add-to-cart conversion | whether proof, fit, and scenario modules reduce purchase hesitation |
| Payment conversion | whether decision confidence carries through checkout |

Main-image funnel metrics:

| funnel step | metric to compare |
|---|---|
| 主图视频 | TTL UV, video engagement, add-to-cart conversion, payment conversion |

Interpretation guardrails:

- Treat post-redesign uplift as supporting evidence, not universal proof.
- If later carousel frames show higher conversion, mention possible self-selection from deeper-intent users.
- Compare like-for-like products, traffic sources, date windows, and promotion conditions before claiming impact.
- Phrase causal claims carefully: "the data supports that function breakdown and style visualization helped parent choice" is safer than "this frame caused the uplift".
